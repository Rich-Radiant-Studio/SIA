User-Web 路由测试

测试说明

1. 确保 user-web 开发服务器正在运行（npm run dev）

2. 访问 http://localhost:8084

3. 打开浏览器控制台（F12）

4. 点击页面上的导航链接

5. 查看控制台日志

常见问题

问题 1: 点击链接没有反应

可能原因：

  • router-view 没有正确渲染
  • 路由没有正确挂载
  • Bootstrap 的 JavaScript 冲突

解决方法：

  • 检查控制台是否有错误
  • 确认看到 "路由配置完成，共 15 个路由"
  • 在控制台输入: window.$router 查看路由实例

问题 2: 页面刷新但没有跳转

可能原因：

  • 路由模式配置问题
  • base URL 不正确

解决方法：

  • 检查 URL 是否变化
  • 尝试手动访问 http://localhost:8084/shop

问题 3: 404 错误

可能原因：

  • 开发服务器不支持 HTML5 History 模式

解决方法：

  • Vite 默认支持，不应该出现此问题
  • 如果出现，检查 vite.config.ts

调试命令

在浏览器控制台执行以下命令：

// 查看所有路由
console.log(window.$router?.getRoutes())
// 测试编程式导航
window.$router?.push('/shop')
// 查看当前路由
console.log(window.$router?.currentRoute.value)
